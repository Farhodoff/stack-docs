# HTML Asoslari va Tuzilma

HTML (HyperText Markup Language) - bu veb-sahifalarni yaratishning asosiy tili. U sahifaning tuzilmasini, matnini, rasmlarini va boshqa kontentni brauzerga qanday ko'rsatish kerakligini belgilaydi.

Bu darsda siz HTML'ning asosiy strukturasini, muhim taglarni va professional web-sahifa yaratishning best practice'larini o'rganasiz.

---

## 🌐 Veb Nima?

Veb - bu **taqsimlangan informatsiya tizimi**, bu erda:
- **Server** - ma'lumotni saqlaydi va yuboradi
- **Internet** - server va foydalanuvchini bog'lash
- **Brauzer** - foydalanuvchining kompyuterida HTML, CSS, JavaScript ko'rsatadi

Qachon siz veb-sahifaga tashrif buyurganizda:
1. Brauzer server'ga so'rov yuboradi
2. Server HTML faylini qaytaradi
3. Brauzer HTML-ni CSS va JavaScript bilan birga ko'rsatadi

---

## 📄 HTML Nima?

**HTML** - bu belgilash tili (markup language). U:
- Matnni, rasmlarni, videoları va boshqa kontentni organize qiladi
- Sahifaning strukturasini belgilaydi
- CSS va JavaScript bilan birgalikda ishlaydi

**Taqqoslash:**
- **HTML** = Uyning suyagi (struktura)
- **CSS** = Uyning terisi (dizayn)
- **JavaScript** = Uyning mushak va nerv sistemasi (xulq-atvor)

## 🏗️ HTML Dokumentining Asosiy Struktura

<CodeBlock language="html" title="HTML Sahifasining Asosiy Tuzilishi">
```html
<!DOCTYPE html>
<html lang="uz">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <meta name="description" content="Sahifaning qisqa tavsifi">
    <title>Sahifa Nomi - Bu Brauzer Tegida Ko'rsatiladi</title>
    <link rel="favicon" href="/favicon.ico">
</head>
<body>
    <!-- Sahifaning asosiy kontent shu yerda bo'ladi -->
    <h1>Xush Kelibsiz!</h1>
    <p>Bu paragraf.</p>
</body>
</html>
```
</CodeBlock>

---

## 1️⃣ <!DOCTYPE html> - Dokumentning Turi

```html
<!DOCTYPE html>
```

Bu **veb-brauzerga** aytadi: "Bu HTML5 dokumenti". Har bir HTML faylining **boshida** bo'lmalı.

**Nima uchun muhim:**
- Brauzer fayl turini bilib oladi
- Standart HTML5 rendering mode'da ishlaydi
- Eski HTML versiyalari (HTML4, XHTML) kabi xatolarga yo'l qo'ymaydi

---

## 2️⃣ <html> - Ildiz Element

```html
<html lang="uz">
    <!-- Butun hujjat shu yerda bo'ladi -->
</html>
```

- **Ildiz element** - HTML dokumentining eng tashqi konteyneri
- `lang="uz"` atributi - sahifaning tilini belgilaydi (Uzbek tili)
- Screen reader'lar va qidiruv tizimlari til ma'lumoti uchun bu atributdan foydalanadi

**Boshqa tillar uchun:**
```html
<html lang="en">      <!-- English -->
<html lang="ru">      <!-- Русский -->
<html lang="ar">      <!-- العربية -->
<html lang="zh">      <!-- 中文 -->
```

## 3️⃣ <head> - Metadata va Konfiguratsiya

`<head>` brauzerga va qidiruv tizimlarga **sahifa haqida ma'lumot** beradi, lekin foydalanuvchi ko'rish uchun ko'rsatilmaydi.

### Meta Taglar (Metadata)

<CodeBlock language="html" title="Muhim Meta Taglar">
```html
<head>
    <!-- Belgilar kodlanishi (UTF-8 - boshqa tillar uchun) -->
    <meta charset="UTF-8">

    <!-- Responsive design uchun viewport metasi -->
    <meta name="viewport" content="width=device-width, initial-scale=1.0">

    <!-- Sahifa tavsifi (Google qidiruva ko'rsatiladi) -->
    <meta name="description" content="Bu veb-sahifaning qisqa tavsifi. 150-160 belgida bo'lsa yaxshi.">

    <!-- Qidiruv tizimlari uchun kalit so'zlar -->
    <meta name="keywords" content="html, css, javascript, web-development">

    <!-- Muallif nomi -->
    <meta name="author" content="Sizning Ismingiz">

    <!-- Copyright -->
    <meta name="copyright" content="© 2026 Sizning Kompaniyasi">

    <!-- Mobile app integrasyon -->
    <meta name="app-mobile-web-app-capable" content="yes">
    <meta name="theme-color" content="#2563eb">

    <!-- Open Graph (Ijtimoiy tarmoqlar uchun) -->
    <meta property="og:title" content="Sahifa Nomi">
    <meta property="og:description" content="Sahifa tavsifi">
    <meta property="og:image" content="/sahifa-rasm.jpg">
    <meta property="og:url" content="https://example.com">
</head>
```
</CodeBlock>

**Nima uchun meta taglar muhim:**
- ✅ Responsive design (mobilda to'g'ri ko'rinish)
- ✅ SEO uchun qidiruv tizimlari sahifani to'g'ri indekslayd
- ✅ Ijtimoiy tarmoqlarda to'g'ri ko'rinish
- ✅ Brauzer xulq-atvori bilan bog'liq shunga o'xshashlar

## 4️⃣ <title> - Sahifa Nomi

```html
<head>
    <title>HTML Darslar - Frontend Asoslari</title>
</head>
```

- Brauzer **tegida** ko'rsatiladi
- Qidiruv tizimlari uchun **muhim** (SEO)
- Foydalanuvchi **browser history**dan sahifani tanlaganda ko'radi

**Best Practice uchun:**
- 30-60 belgida bo'lsa yaxshi
- Kalit so'z avval, kompaniya nomi oxirida
- Misal: `"HTML Darslar - Fullstack Docs"` - yaxshi ✅

---

## 5️⃣ Favicon - Sahifa Belgisi

```html
<head>
    <link rel="icon" href="/favicon.ico">
    <!-- yoki PNG formatida -->
    <link rel="icon" type="image/png" href="/favicon.png">
</head>
```

- Brauzer **tegida** va address bar'da ko'rsatiladi
- Foydalanuvchiga sahifani tanish qiladi
- 16x16 yoki 32x32 piksel o'lchamda

---

## 6️⃣ <body> - Sahifaning Asosiy Kontent

```html
<body>
    <!-- Foydalanuvchi ko'rishi kerak bo'lgan hamma kontent shu yerda bo'ladi -->
    <h1>Sarlavha</h1>
    <p>Paragraf matn</p>
    <img src="/rasm.jpg" alt="Rasm tavsifi">
</body>
```

- **Faqat bitta `<body>`** har bir HTML faylida
- Brauzerda ko'rinadi (Head'dan farqli)
- Barcha mashhur elementlar `<body>` ichida bo'ladi

## 📋 To'liq HTML5 Sahifasi Misoli

<CodeBlock language="html" title="Profesional HTML5 Sahifasi">
```html
<!DOCTYPE html>
<html lang="uz">
<head>
    <!-- Meta taglar va kodlanish -->
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <meta name="description" content="Frontend dasturlashni o'rganish uchun to'liq qo'llanma.">
    <meta name="keywords" content="html, css, javascript, web">
    <meta name="author" content="Fullstack Docs">

    <!-- Sahifa nomi -->
    <title>HTML Darslar - Frontend Asoslari | Fullstack Docs</title>

    <!-- Favicon -->
    <link rel="icon" href="/favicon.ico">

    <!-- CSS -->
    <link rel="stylesheet" href="/styles/main.css">

    <!-- Open Graph ijtimoiy tarmoqlar uchun -->
    <meta property="og:title" content="HTML Darslar">
    <meta property="og:description" content="Frontend dasturlashni o'rganish">
    <meta property="og:url" content="https://example.com/html">
</head>
<body>
    <!-- Sahifa sarlavhasi -->
    <header>
        <h1>🚀 Frontend Darslar</h1>
        <p>HTML, CSS, va JavaScript'ni o'rganin</p>
    </header>

    <!-- Navigatsiya -->
    <nav>
        <ul>
            <li><a href="/">Bosh sahifa</a></li>
            <li><a href="/darslar">Darslar</a></li>
            <li><a href="/loyihalar">Loyihalar</a></li>
            <li><a href="/haqida">Haqida</a></li>
        </ul>
    </nav>

    <!-- Asosiy kontent -->
    <main>
        <section>
            <h2>Xush Kelibsiz!</h2>
            <p>Bu darsda siz HTML'ning asoslari haqida o'rganasiz.</p>
        </section>
    </main>

    <!-- Pastki qism -->
    <footer>
        <p>&copy; 2026 Fullstack Docs. Barcha huquqlar himoyalangan.</p>
    </footer>

    <!-- JavaScript -->
    <script src="/scripts/main.js"></script>
</body>
</html>
```
</CodeBlock>

## 🎯 Semantik Dokumentning Tuzilishi

**Semantik HTML** - elementlarning mantiqiy ma'nosiga qarab to'g'ri ishlatish:

```html
<body>
    <!-- Sayt sarlavhasi va logo -->
    <header>
        <h1>Sayt Nomi</h1>
        <nav><!-- Menu --></nav>
    </header>

    <!-- Asosiy kontent -->
    <main>
        <!-- Bo'limlar -->
        <section>
            <h2>Bo'lim 1</h2>
            <article><!-- Maqola --></article>
        </section>
    </main>

    <!-- Yon panel -->
    <aside>
        <!-- Qo'shimcha ma'lumot -->
    </aside>

    <!-- Pastki qism -->
    <footer>
        <p>Mualliflik huquqi</p>
    </footer>
</body>
```

**Nima uchun muhim:**
- ✅ Screen reader'lar strukturani tushunadi
- ✅ Qidiruv tizimlari sahifani o'nglishni aniqroq qiladi
- ✅ Kodni boshqa dasturchilar oson o'qiydi
- ✅ Maintenance va tahrirlash osonroq

## ✅ Best Practices

### 1. Har Doim UTF-8 Belgilardan Foydalaning

```html
<meta charset="UTF-8">
```

Bu **o'zbekcha harflar**, emoji va boshqa belgilar to'g'ri ko'rinishini ta'minlaydi.

### 2. Responsive Viewport Metasini Ishlatish

```html
<meta name="viewport" content="width=device-width, initial-scale=1.0">
```

Bu **mobil qurilmalar** uchun sahifani to'g'ri o'lchashni ta'minlaydi.

### 3. Semantik Taglarni Ishlatish

```html
<!-- ✅ Yaxshi -->
<header>
    <nav><!-- Menu --></nav>
</header>
<main>
    <article><!-- Kontent --></article>
</main>
<footer><!-- Pastki qism --></footer>

<!-- ❌ Yomon -->
<div id="header">
    <div id="nav"><!-- Menu --></div>
</div>
```

### 4. Katta Head'dan Foydalanmang

```html
<!-- ❌ Yomon -->
<h1>Sarlavha 1</h1>
<h3>Sarlavha 3</h3>  <!-- h2 skip qilindi! -->

<!-- ✅ Yaxshi -->
<h1>Sarlavha 1</h1>
<h2>Sarlavha 2</h2>
<h3>Sarlavha 3</h3>
```

### 5. Alt Atributini Rasmlar Uchun Ishlatish

```html
<!-- ❌ Yomon -->
<img src="/rasm.jpg">

<!-- ✅ Yaxshi -->
<img src="/rasm.jpg" alt="Yoshlar birgalikda dasturlash qilyapti">
```

### 6. Aniqlashni Ishlatish

```html
<!-- ❌ Yomon -->
<link rel="stylesheet" href="styles.css">

<!-- ✅ Yaxshi -->
<link rel="stylesheet" href="/css/styles.css">
```

## ❌ Keng Uchraydigan Xatoliklar

### 1. DOCTYPE Yoʻqligi

```html
<!-- ❌ Yomon - DOCTYPE yoʻq -->
<html>
    <head>
        <title>Mening Saytim</title>
    </head>
</html>

<!-- ✅ Yaxshi - DOCTYPE bor -->
<!DOCTYPE html>
<html lang="uz">
    <head>
        <title>Mening Saytim</title>
    </head>
</html>
```

### 2. Til Atributini Belgilamamasliq

```html
<!-- ❌ Yomon -->
<html>

<!-- ✅ Yaxshi -->
<html lang="uz">
```

### 3. Closing Taglarni Tashlab Qolish

```html
<!-- ❌ Yomon -->
<body>
    <h1>Sarlavha
    <p>Paragraf

<!-- ✅ Yaxshi -->
<body>
    <h1>Sarlavha</h1>
    <p>Paragraf</p>
</body>
```

### 4. Viewport Metasini Belgilamamasliq

```html
<!-- ❌ Mobilda mobilda yomon ko'rinadi -->
<meta charset="UTF-8">

<!-- ✅ Mobilda to'g'ri ko'rinadi -->
<meta charset="UTF-8">
<meta name="viewport" content="width=device-width, initial-scale=1.0">
```
