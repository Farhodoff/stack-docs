# Semantik HTML va Accessibility

**Accessibility (A11y)** - web-sahifalarni **barcha foydalanuvchilar** uchun, shu jumladan **o'ziga xos ehtiyojlari bo'lgan** odamlar uchun ishlashi. Bu darsda siz semantik HTML va accessibility'ni o'rganasiz.

---

## 1️⃣ Semantik HTML Nima?

**Semantik HTML** - elementlarning **ma'nosiga qarab** to'g'ri ishlatish.

### Semantik vs Non-Semantik

```html
<!-- ❌ Non-Semantik - ma'nosi yo'q -->
<div class="header">Sayt Nomi</div>
<div class="nav">Menu</div>
<div class="main">Kontent</div>
<div class="footer">Copyright</div>

<!-- ✅ Semantik - ma'nosi bor -->
<header>Sayt Nomi</header>
<nav>Menu</nav>
<main>Kontent</main>
<footer>Copyright</footer>
```

### Nima Uchun Semantik HTML?

| Faida | Misol |
|--------|-------|
| **SEO** | Google sahifaning strukturasini to'g'ri tushunadi |
| **Accessibility** | Screen reader'lar navigatsiyani biladi |
| **Kod Kalitesi** | Dasturchilar oson tushunadi |
| **Mobile** | O'zbekcha brauzerlar to'g'ri render qiladi |

## 5️⃣ ARIA Atributlari

**ARIA (Accessible Rich Internet Applications)** - screen reader'lar uchun qo'shimcha ma'lumot beradigan atributlar.

### aria-label - Element Nomi

```html
<!-- Screen reader: "Qidirish tugmasi" deb o'qiydi -->
<button aria-label="Qidirish">🔍</button>

<!-- Screen reader: "Menu ochish tugmasi" -->
<button aria-label="Menu ochish">☰</button>
```

### aria-describedby - Batafsil Tavsif

```html
<input type="password" id="pwd" aria-describedby="pwd-hint">
<p id="pwd-hint">Parol kamida 8 ta belgi bo'lishi kerak</p>

<!-- Screen reader: input tavsifi va hint'ni o'qiydi -->
```

### aria-live - Real-time Yangilanishlar

```html
<!-- Real-time bildirishnomalar (chat, alerts) -->
<div aria-live="polite" role="status">
    Ali: Salom! Bu yerga yangi xabar
</div>

<!-- aria-live="assertive" - juda muhimni darhol o'qiydi -->
<div aria-live="assertive" role="alert">
    XATO: Forma yuborilmadi!
</div>
```

### aria-hidden - Screen Reader'dan Yashiring

```html
<!-- Dekorativ elementlar - accessibility uchun yashiring -->
<span aria-hidden="true">✨</span> Mazali Kontent

<!-- Yashirun matn (CSS: display: none) -->
<div aria-hidden="true">This is hidden</div>
```

## ✅ Accessibility Best Practices

### 1. Alt Tavsifini Har Doim Beya

```html
<!-- ❌ Yomon -->
<img src="/photo.jpg">

<!-- ✅ Yaxshi -->
<img src="/photo.jpg" alt="Toshkent shahri ko'li">
```

### 2. Semantik Taglarni Ishlatish

```html
<!-- ❌ Yomon -->
<div class="header">...</div>
<div class="nav">...</div>

<!-- ✅ Yaxshi -->
<header>...</header>
<nav>...</nav>
```

### 3. Color Ga Ta'yanmang

```html
<!-- ❌ Yomon - faqat rang (rang ko'rishing muammosi) -->
<p style="color: red;">XATO</p>

<!-- ✅ Yaxshi - rang + tekst -->
<p style="color: red;"><strong>❌ XATO:</strong> ...</p>
```

### 4. Label Bilan Forma Inputlari

```html
<!-- ❌ Yomon -->
<input type="text">

<!-- ✅ Yaxshi -->
<label for="username">Username:</label>
<input type="text" id="username">
```

### 5. Keyboard Navigatsiyasi

```html
<!-- ❌ Yomon - faqat sichqoncha -->
<div onclick="doSomething()">Bosing</div>

<!-- ✅ Yaxshi - keyboard ham -->
<button onclick="doSomething()">Bosing</button>
```

## 📱 Responsive Jadval (CSS bilan)

        }

            padding-left: 50%;
